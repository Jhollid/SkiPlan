# MVP Scope

The first release is intentionally constrained to the route-planning core:

- Single supported resort: `Flims Laax`
- Email/password authentication only
- Saved skiing preferences
- Resort list and map metadata
- Step-by-step route planning between resort nodes
- Lift open/closed support via manual flags or provider adapters later

Explicitly deferred until after the MVP:

- Groups and location sharing
- Venue discovery and advertising
- Offline static package generation
- Multi-resort support
